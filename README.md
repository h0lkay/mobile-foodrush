# Mobile Food Rush

## Сборка проекта

### Требования
- Android Studio
- JDK 11 или выше
- Android SDK
- Gradle 7.0+

### Инструкция по сборке

#### 1. Клонирование репозитория

```bash
git clone https://github.com/h0lkay/mobile-foodrush.git
cd mobile-foodrush
```

#### 2. Сборка APK

Через Android Studio:

Build → Build Bundle(s) / APK(s) → Build APK(s)

Готовый APK будет в app/build/outputs/apk/debug/

Через командную строку:

```bash
./gradlew assembleDebug
```

## Руководство пользователя

### 1. Главный экран (Погода)

При запуске приложения открывается главный экран, на котором отображается актуальная информация о погоде.
• Просмотр данных: На экране отображается город (по умолчанию Красноярск), текущая температура в градусах Цельсия и иконка, описывающая погодные условия (например, "ясно" или "небольшой дождь").
• Обновление данных: Данные подгружаются автоматически при запуске из сервиса OpenWeatherMap.

### 2. Работа с картой и выбор адреса

Раздел карты на странице добавления нового адреса доставки позволяет визуально определить местоположение и ввести данные для доставки.

• Интерактивная карта: В приложении используется карта OpenStreetMap. Вы можете масштабировать её и перемещаться по ней для поиска нужного объекта.
• Установка метки: На странице списка адресов, на карте установлены метки, при нажатии на которые появляется всплывающее окно с описанием объекта.
• Ввод адреса: В нижней части экрана расположена панель "Адрес доставки":
i.
При добавлении метки в поле улица и дом автоматически добавляется информация
ii.
Укажите номер квартиры и подъезда в соответствующих полях.
iii.
При необходимости уточните город.
• Сохранение: После ввода данных нажмите кнопку «Доставить сюда». Система зафиксирует введенный адрес для дальнейшего оформления заказа.
• Закрытие: Чтобы вернуться на предыдущий экран без сохранения, нажмите на иконку «крестик» в верхнем левом углу.

## План монетизации
Поскольку приложение является сервисом по доставке еды, наиболее эффективными будут комбинированные модели:
### Модель комиссий (Основная)
- Комиссия с заказов: Взимание 10–15% с каждого чека ресторана, исходя из суммы заказа;
- Сервисный сбор: Небольшая фиксированная плата с пользователя (например, 29–49 ₽) за «поддержку и развитие сервиса»;
### Рекламные возможности
- Приоритетное размещение: Ресторан платит за то, чтобы быть первым в списке на главной странице или в результатах поиска.
- Брендированные метки на карте: Оформление метки ресторана на карте в особенном стиле.
- Push-уведомления: Рассылка рекламных предложений ресторана пользователям в определенном радиусе.
### Премиум-функции
- Подписка «Красноярск Плюс»:
  - Бесплатная доставка при заказе от определенной суммы.
  - Приоритетное обслуживание в часы пик.

## План публикации

### Шаг 1: Техническая оптимизация

- Обфускация кода: Настройка ProGuard/R8 для защиты от реверс-инжиниринга.
- Сжатие ресурсов: Оптимизация иконок и фоновых изображений (webp) для уменьшения веса APK/AAB.
- Обработка ошибок: Проверка всех try-catch блоков, чтобы приложение не вылетало при отсутствии интернета или неверном API-ключе.

### Шаг 2: Площадки для публикации

Учитывая текущую ситуацию и локализацию приложения под РФ (Красноярск), план следующий:
1. RuStore (Приоритет №1): Главный магазин приложений в РФ. Обязателен для охвата аудитории, использующей российские банковские приложения.
2. Google Play Store: Для доступа ко всем Android-пользователям.
3. Huawei AppGallery: Важен для владельцев устройств Huawei/Honor без Google-сервисов.

### Шаг 3: План запуска по этапам

| Этап | Действие | Цель |
| :--- | :--- | :--- |
| **I. Alpha-тест** | Закрытое тестирование среди своих (5–10 человек) | Поиск багов в логике корзины и карты |
| **II. Beta-тест** | Публикация в RuStore/Google Play в режиме "Beta" | Сбор первых отзывов жителей Красноярска |
| **III. Soft Launch** | Запуск рекламы на 1 район города | Проверка нагрузки на сервер и работу курьеров |
| **IV. Full Release** | Полноценный запуск по всему городу | Масштабирование и начало активной монетизации |

## Средства обеспечения безопасности приложения

В приложении реализованы следующие уровни защиты:
1. Безопасное хранение API-ключей: Ключ погоды вынесен в gradle.properties и подключается через BuildConfig. Это предотвращает попадание секретных данных в публичные репозитории.
2. Сетевая безопасность: В файле app/src/main/res/xml/network_security_config.xml реализована строгая политика. Параметр cleartextTrafficPermitted="true" ограничен только базовым уровнем для совместимости с тайлами OpenStreetMap (OSM). Весь остальной API-трафик (OpenWeatherMap) принудительно переведен на TLS/SSL (HTTPS).
3. Безопасность данных (SQLite): Работа с базой данных через DatabaseHelper и ContentValues предотвращает большинство атак типа SQL-инъекция, так как данные в запросах автоматически экранируются системой.
4. Изоляция данных: Приложение использует систему разрешений Android. Например, доступ к интернету и местоположению запрашивается явно, а база данных хранится во внутреннем хранилище приложения, недоступном для других программ.
5. База данных AppDatabase.db находится в защищенном приватном хранилище приложения, недоступном для стороннего софта без Root-прав.
